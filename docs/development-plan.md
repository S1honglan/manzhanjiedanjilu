# 开发执行计划

## 总览

本项目采用分阶段开发方式，每个阶段有明确目标和验收标准。完成一个阶段并验证通过后，再进入下一阶段。

当前范围：第一期（核心 CRUD + 搜索筛选）

---

## 阶段 1：重构数据模型与基础架构

### 目标
根据新需求重整数据结构，建立「漫展项目 → 客户订单」的两层数据关系。

### 具体步骤

1. **重新设计数据类型**（`src/types.ts`）
   - `ExpoProject`：`id`、`name`（漫展名称）、`date`（漫展日期）、订单列表、时间戳
   - `ClientOrder`：`id`、`character`（角色）、`customerName`（昵称）、`photoCount`（张数）、`price`（全款价格）、`deposit`（定金）、`totalIncome`（全部收入）、`status`（订单状态）、`note`（备注）、所属项目ID、时间戳
   - `OrderStatus`：保持 `'预付定金' | '工作中' | '交付'`

2. **重写存储层**（`src/storage.ts`）
   - 改为存储 `ExpoProject[]`，每个项目内嵌 `ClientOrder[]`
   - 提供读/写全部项目的方法

3. **清理旧代码**
   - 移除 `src/data.ts` 中的旧示例数据（或改写为新结构示例）
   - 移除旧的 Excel 导入导出相关文件（本期不做）
   - 移除 `src/excel.ts`

4. **验证方式**：启动 dev server，确认无编译错误，localStorage 中可看到新数据结构

---

## 阶段 2：漫展项目卡片管理

### 目标
实现首页的项目卡片列表，支持新建、编辑、删除漫展项目。

### 具体步骤

1. **项目卡片组件**（`src/components/ProjectCard.tsx`）
   - 显示：漫展名称、漫展日期、内含订单数
   - 悬停显示编辑/删除按钮
   - 点击进入项目内页

2. **新建项目表单**（模态框或底部面板）
   - 字段：漫展名称（必填）、漫展日期（日期选择器）
   - 确认/取消按钮

3. **编辑项目功能**
   - 点击项目卡片上的编辑按钮弹出编辑表单
   - 可修改名称和日期

4. **删除项目（带确认）**
   - 点击删除按钮弹出确认提示
   - 确认后删除项目及其下所有订单

5. **首页布局**
   - 电脑端：卡片网格（2-3 列）
   - 手机端：单列卡片列表

6. **验证方式**：可新建 2-3 个项目，编辑项目名称，删除一个项目，刷新后数据仍在

---

## 阶段 3：客户订单卡片管理

### 目标
进入项目后展示该项目下的客户订单列表，支持新建、编辑、删除订单。

### 具体步骤

1. **订单卡片组件**（`src/components/OrderCard.tsx`）
   - **角色名**用大号字体，作为卡片最醒目的标题
   - 小字显示：客户昵称、拍摄张数、全款价格、定金、全部收入
   - 订单状态用颜色标签显示
   - 备注可折叠/展开

2. **新建订单表单**
   - 字段与需求一致，角色与客户昵称为必填
   - 订单状态默认为「预付定金」

3. **编辑订单功能**
   - 点击订单卡片弹出编辑面板
   - 可修改所有字段

4. **删除订单功能**
   - 带确认提示

5. **验证方式**：在项目下新建订单，编辑订单内容，删除订单，确认数据正确保存

---

## 阶段 4：搜索筛选 + UI 完善

### 目标
实现搜索和筛选功能，完善响应式布局和视觉细节。

### 具体步骤

1. **搜索功能**
   - 搜索框实时过滤：按角色名、客户昵称搜索
   - 输入即过滤，无需按回车

2. **筛选功能**
   - 按订单状态筛选（全部/预付定金/工作中/交付）
   - 按漫展日期区间筛选
   - 筛选器在手机端可折叠

3. **视觉细节完善**
   - 按设计规范统一配色（淡蓝主色 + 状态颜色标签）
   - 卡片圆角、阴影、间距微调
   - 响应式断点：768px 为分界

4. **验证方式**：在桌面和手机浏览器（或手机模拟器）测试全部功能正常

---

## 阶段 5：整体测试与打磨

### 目标
整体回归测试，修复问题，完善体验。

### 具体步骤

1. **手动回归测试**
   - 新建/编辑/删除项目
   - 新建/编辑/删除订单
   - 搜索筛选各功能正常
   - 数据刷新后不丢失

2. **响应式测试**
   - 桌面端（>768px）：卡片网格正常
   - 手机端（≤768px）：单列卡片正常，操作方便

3. **整理文档**
   - 补充开发日志
   - 如需要，更新各规范文档
