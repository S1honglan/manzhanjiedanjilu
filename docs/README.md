# 项目文档说明

本目录包含项目开发过程中需要维护的标准文档。

## 文档清单

- `requirements.md`：项目需求说明、两层卡片结构、字段定义、功能范围。
- `technical-spec.md`：技术选型、数据结构、组件树、存储方案、响应式策略。
- `design-guidelines.md`：视觉风格、配色规范、排版层级、卡片/按钮/布局规范。
- `development-standards.md`：开发流程、版本控制、代码规范、测试标准、日志规范。
- `development-plan.md`：分阶段开发执行计划，详细步骤与进度追踪。

## 使用说明

1. 开发前先阅读 `requirements.md`，确认功能和数据结构。
2. 进入开发前确认 `technical-spec.md` 中的技术方案和实现要点。
3. 参考 `design-guidelines.md` 实现样式和交互。
4. 按照 `development-plan.md` 的阶段顺序推进，完成一个阶段再进入下一个。
5. 每次开发完成后，若有流程或规范变更，及时同步到对应文档。
