# 项目需求说明

## 项目目标

开发一个「漫展拍摄接单记录」应用，可以在电脑和手机浏览器上使用。

## 核心结构：两层卡片

```
📁 漫展项目卡片（一场漫展）
  ├── 👤 客户订单卡片1（角色A · 客户张三）
  ├── 👤 客户订单卡片2（角色B · 客户李四）
  └── 👤 客户订单卡片3（角色C · 客户王五）
```

- **项目卡片** 代表一场漫展，包含这场漫展的所有客户订单。
- **客户订单卡片** 嵌在项目卡片内，记录单个客户的拍摄信息。

---

## 漫展项目卡片 包含的信息

| 字段 | 说明 |
|------|------|
| 漫展名称 | 比如「CP30」「萤火虫漫展」 |
| 漫展日期 | 漫展举办的日期 |

---

## 客户订单卡片 包含的信息

| 字段 | 说明 | 卡片显示 |
|------|------|----------|
| 拍摄角色 | 客户要拍摄的角色名 | **大标题，最醒目** |
| 客户昵称 | 客户的昵称或称呼 | 小字 |
| 拍摄张数 | 拍摄多少张照片 | 小字 |
| 全款价格 | 这一单的总价 | 小字 |
| 定金 | 客户已经付了多少定金 | 小字 |
| 全部收入 | 这个订单实际收到的总收入 | 小字 |
| 备注 | 额外备注 | 小字/可折叠 |
| 订单状态 | 预付定金 / 工作中 / 交付 | 彩色标签 |

### 订单状态颜色

- `预付定金`：淡蓝色标签
- `工作中`：淡橙色标签
- `交付`：淡绿色标签

---

## 页面结构

1. **首页**：显示所有漫展项目卡片
2. **项目内页**：点击项目卡片后，展示该项目下所有客户订单卡片
3. **新建/编辑**：可以新建或编辑项目卡片和客户订单卡片

---

## 搜索与筛选

- **搜索**：支持按角色名、客户昵称搜索
- **筛选**：支持按订单状态、漫展日期筛选

---

## 视觉风格

- **风格**：简洁、直观、低饱和色系
- **主色**：淡蓝色
- **卡片**：圆角、轻微阴影、白色背景
- **文字**：角色名大标题醒目，其他信息用较小字号

## 技术要点

- 数据保存在浏览器本地（localStorage），无需后台服务器
- 电脑端多列卡片布局，手机端单列布局
- 所有数据离线可用
